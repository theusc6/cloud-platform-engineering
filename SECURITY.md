# Security Policy

## Reporting a Vulnerability

If you have discovered a security vulnerability in this project, we appreciate your help in disclosing it to us in a responsible manner.

Please do the following:

* **Do not** post the vulnerability publicly (e.g. on social media, your personal blog, or in a public issue tracker).
* **Do not** attempt to exploit the vulnerability for any reason.
* Open a [security advisory](https://github.com/theusc6/cloud-platform-engineering/security/advisories/new) on this repository with a description of the vulnerability and any related information (e.g. steps to reproduce the issue).
* If possible, provide a proposed solution for the vulnerability.

We will make every effort to respond to your email within 48 hours and to address any vulnerabilities as soon as possible.

### Disclaimer

While we make every effort to ensure the security of this project, please be aware that no method of transmission over the internet, or method of electronic storage, is 100% secure. We cannot guarantee the absolute security of your personal information or the information contained in this project.

_By using this project, you acknowledge and accept these security risks._
