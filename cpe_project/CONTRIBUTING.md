# Contributing to cpe_project

Thank you for your interest in contributing to the `cpe_project` repository! We appreciate your efforts to make this project even better. Before you start, please take a moment to review these guidelines.

## Getting Started

1. Clone the repository on GitHub.
2. Create a new branch for your work: `git checkout -b feature/your-feature` or `git checkout -b bugfix/fix-bug`.
3. Make your changes and test them thoroughly.
4. Commit your changes with clear and descriptive commit messages.
5. Push your changes to your remote branch: `git push --set-upstream origin feature/new-feature`.
6. Open a pull request (PR) to the `main` branch of the `cpe_project` repository.

## Pull Request Guidelines

- Ensure your changes are well-tested and do not break existing functionality.
- You are encouraged to use local linting tools; e.g. `pylint` before committing.
- Keep your pull requests focused on a single feature or bug fix.
- Use descriptive titles and provide detailed descriptions of your changes.
- Reference any related issues using keywords like "Fixes #123" or "Closes #456."

## Code Review

- All pull requests will be reviewed by maintainers; e.g. `CODEOWNERS`
- Address any feedback or comments provided during the review process.
- If there are any requested changes, make the necessary updates and notify the reviewer.

## `main` Branch Protections

The `main` branch is protected to ensure code quality and stability. This means:

- Direct pushes to `main` are not allowed.
- Changes must be made through pull requests.
- Code reviews and status checks must pass before merging.

## CODEOWNERS for Approvals

We have CODEOWNERS set up to facilitate the review and approval process. Code owners are responsible for reviewing and approving changes related to specific parts of the repository.

If your pull request modifies code owned by specific individuals or teams, please request their review by mentioning them in your pull request description or using the "Reviewers" feature in the GitHub UI.
